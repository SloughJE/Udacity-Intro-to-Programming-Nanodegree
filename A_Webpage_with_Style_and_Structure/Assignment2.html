Lesson 1.2: Creating a Structured Document with HTML

Boxes

Everything in HTML is a box, even circles. An imaginary box surrounds each section or element of the webpage.
Start thinking in boxes to design a webpage.

Tree-like Structure



Webpages have a tree-like structure (branching structure). I am familar with this concept from the study of decision trees.

This appears in HTML as sideways triangles (HTML elements). Each has an opening and closing tag.

There are parent and child elements, i.e. a root, containing branches, containing more branches...

This screenshot is an example of the tree-like, parent-child relationship of HTML elements from my code. From the body element, if we consider that the parent, the div element below that would be the child. This relationship can be clearly seen in the indentation, especially with that of lists as shown here. The elements, corresponding to the triangles, can be opened and closed to navigate the tree.

The HTML CSS and DOM Relationship

HTML is the language of the web which has syntax and rules.
Each 'word' in HTML is a tag, and the browswer turns the tags into elements in a tree.
It knows how to do this because of DOM.
DOM or Document Object Model represents and allows interaction with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree. This is the tree seen in developer tools.
An HTML tag creates an element in the DOM that the browser uses to display the page.
Element: 
<starting_tag> content </ending_tag>
Each tag can have an attribute with a value. For example: 
<h1 class="article">
h1 is the tag type, with a class attribute, with a value of article.

CSS defines the style. This allows you to use specific syntax and rules to change how elements look on the page.
You can use CSS to say all h1 elements that belong to the class article should have the text color blue.


Lesson 1.3: An HTML Page with Style and Structure

CSS

  • Cascading Style Sheets
    • add style to the HTML document (fonts, colors, spacing...etc
    • can be added in the head of the HTML document or in a separate CSS file
    • must be in same folder

Here are some examples of code to add style to the HTML document:

The code: 

h1 {
  text-align: center;
  color: #294860;
}
makes all h1 headers centered and of the hexidecimal color #294860.

The code:
<div class="lesson">
references the style rule which corresponds to this class.

Here is an example corresponding style rule which would go in the style section or file:
.lesson {
background-color: #0F7F9A;
border-radius: 20px;
color: white;
padding: 10px;
margin: 10px;
}
There are a multitude of style rules so just look them up or use the developer tools in Chrome to fine tune the style.

Note: do not add styling inline with the HTML code. This will lead to many problems.

Avoiding Repitition

A big idea of programming is avoiding repitition. The proper use of CSS is an expression of this.
As discussed above, I can use an HTML class attribute, and link it to a specific style in my CSS file. I can set all h1 headers to display in blue text with one line of code in the CSS file. This prevents me from having to type it manually for each and ever h1 header. I can also do this by assigning a class to certain elements that I want to be displayed in a specific way. For example, I can have the element 
<div class="lesson">
be linked to the style 
.lesson {
  background-color: #0f7f9a;
  border-radius: 20px;
  color: white;
  padding: 10px;
  margin: 10px;
}
which is exactly what I've done in this document. Now all the elements that I give the class lesson to have the same style.
As you can imagine, it saves time, prevents errors, and makes it easier to develop and maintain code. It also helps the webpage look like it has its own style.
